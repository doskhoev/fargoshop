# FargoShop - Интернет-магазин продуктов питания

Современный полнофункциональный интернет-магазин продуктов питания с системой аутентификации, админ-панелью и интеграцией с базой данных PostgreSQL.

## 🚀 Функциональность

### Для покупателей:
- 🛒 **Каталог товаров** с фильтрацией по категориям
- 🔍 **Поиск товаров** по названию и описанию
- 🛍️ **Корзина покупок** с управлением количеством (+/-)
- 📦 **Оформление заказа** с указанием телефона и адреса доставки
- 💳 **Оплата при получении** (задел под ЮKassa)
- 📋 **История заказов** и отслеживание статуса
- 📱 **Адаптивный дизайн** для всех устройств
- 👤 **Регистрация и авторизация** (телефон обязателен, адрес — опционально)
- 💾 **Корзина** — на сервере для авторизованных, localStorage для гостей (merge при входе)
- 👤 **Редактирование профиля** — `/profile` (имя, email, телефон, адрес, смена пароля)

### Для администраторов:
- 🔐 **Система ролей** (USER / ADMIN / PICKER / COURIER)
- ➕ **Добавление новых товаров**
- ✏️ **Редактирование существующих товаров**
- 🗑️ **Удаление товаров**
- 📊 **Управление всеми заказами**
- 👥 **Список пользователей и создание аккаунтов сборщиков/курьеров**

### Для сборщиков и курьеров:
- 📦 **Сборщик** — заказы на сборку (`CONFIRMED` → `ASSEMBLING` → `READY_FOR_DELIVERY`)
- 🚚 **Курьер** — заказы к доставке (`READY_FOR_DELIVERY` → `IN_DELIVERY` → `DELIVERED`)

## 🛠️ Технологический стек

### Frontend:
- **Next.js 14** - React фреймворк с App Router
- **TypeScript** - типизированный JavaScript
- **Tailwind CSS** - utility-first CSS фреймворк
- **Zustand** - легковесное управление состоянием
- **Heroicons** - иконки от создателей Tailwind

### Backend:
- **Next.js API Routes** - серверные маршруты
- **PostgreSQL** - реляционная база данных
- **Prisma** - современная ORM для TypeScript
- **JWT** - аутентификация через токены
- **bcryptjs** - хеширование паролей
- **Zod** - валидация данных

### Инструменты разработки:
- **Yarn** - пакетный менеджер
- **ESLint** - линтер для JavaScript/TypeScript
- **Prisma Studio** - GUI для работы с базой данных

## 📦 Установка и запуск

### 1. Клонирование и установка зависимостей
```bash
git clone <repository-url>
cd fargoShop
yarn install
```

### 2. Настройка базы данных
```bash
# Создайте файл .env в корне проекта
echo "DATABASE_URL=\"postgresql://username:password@localhost:5432/fargoshop\"" > .env

# Примените схему базы данных
yarn db:push

# Создайте тестовые данные
yarn db:seed
```

### 3. Запуск приложения
```bash
# Режим разработки
yarn dev

# Приложение будет доступно на http://localhost:3000
```

## 🗄️ Команды для работы с базой данных

```bash
# Генерация Prisma клиента
```

## 🔧 API Endpoints

### Аутентификация:
- `POST /api/auth/login` - вход в систему
- `POST /api/auth/register` - регистрация (телефон обязателен, адрес — опционально)
- `GET /api/auth/me` - получение данных пользователя
- `PATCH /api/auth/profile` - обновление профиля и смена пароля

### Корзина (требуется авторизация):
- `GET /api/cart` - получить корзину
- `POST /api/cart` - добавить товар `{ productId, quantity? }`
- `PUT /api/cart` - установить количество `{ productId, quantity }`
- `DELETE /api/cart` - очистить корзину (`?productId=` — удалить позицию)

### Товары:
- `GET /api/products` - список товаров (с фильтрами)
- `POST /api/products` - создание товара (только админ)
- `PUT /api/products/[id]` - обновление товара (только админ)
- `DELETE /api/products/[id]` - удаление товара (только админ)

### Заказы:
- `POST /api/orders` - создание заказа (гость или авторизованный)
- `GET /api/orders` - список заказов текущего пользователя
- `GET /api/orders/[id]` - детали заказа
- `PATCH /api/orders/[id]/status` - смена статуса (staff)
- `GET /api/orders/staff` - заказы для сборщика/курьера

### Админ:
- `GET /api/admin/orders` - все заказы
- `GET /api/admin/users` - список пользователей
- `POST /api/admin/users` - создание аккаунта сборщика/курьера

### Платежи:
- `POST /api/payments/webhook/yukassa` - webhook ЮKassa (заглушка)
